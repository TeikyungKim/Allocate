## 2. 구글 로그인 실패 (`DEVELOPER_ERROR`)

### 원인
`google-services.json`과 `AuthContext.tsx`의 `webClientId`가 **placeholder 값**인 상태입니다.

```
webClientId: '000000000000-placeholder.apps.googleusercontent.com'
project_id: 'allocate-placeholder'
```

### 해결 방법

#### Step 1: Firebase 프로젝트 생성
1. [Firebase Console](https://console.firebase.google.com/) 접속
2. 새 프로젝트 생성 (예: `allocate-app`)
3. Android 앱 추가: 패키지명 `com.allocate.app`

#### Step 2: SHA-1 키 등록 (핵심!)
```bash
# 디버그 키스토어 SHA-1 확인
keytool -list -v -keystore ~/.android/debug.keystore -alias androiddebugkey -storepass android

# EAS 빌드 키스토어 SHA-1도 등록 필요
eas credentials -p android
```

- Firebase Console → 프로젝트 설정 → Android 앱 → SHA 인증서 지문 추가
- **디버그 SHA-1**과 **릴리스 SHA-1** 모두 등록해야 함

#### Step 3: google-services.json 교체
1. Firebase Console에서 `google-services.json` 다운로드
2. 프로젝트 루트의 `google-services.json` 교체

#### Step 4: webClientId 업데이트
1. Firebase Console → Authentication → Sign-in method → Google 활성화
2. Web client ID 복사
3. `src/contexts/AuthContext.tsx`의 `webClientId` 값 교체

#### Step 5: EAS 빌드 재실행
```bash
eas build --profile development --platform android
```

> **참고**: 개발 빌드(`--profile development`)에서도 Google 로그인은 정상 동작해야 합니다. 문제는 프로필이 아니라 Firebase 설정 미완성입니다.

## 3. 광고가 안 나오는 문제

### 현재 동작 방식
- `__DEV__` 모드(개발 빌드)에서는 Google AdMob **테스트 광고 ID** 사용
- `TestIds.BANNER`, `TestIds.INTERSTITIAL`로 자동 전환

### 왜 안 나올 수 있는가?
1. **개발 빌드 첫 실행**: AdMob SDK 초기화에 시간이 걸릴 수 있음 (1-2분)
2. **네트워크 문제**: 테스트 광고도 Google 서버 연결 필요
3. **AdMob 앱 등록 미완료**: `app.json`의 `react-native-google-mobile-ads`에 실제 앱 ID가 필요
   - 현재 `ca-app-pub-9164387280735910~3734498758` (Android)로 설정되어 있음
   - 이 앱 ID가 AdMob Console에 등록되어 있어야 함
4. **에뮬레이터/실기기 차이**: 일부 에뮬레이터에서 광고 로드 실패

### 확인 방법
- 앱 로그에서 `AdMob` 관련 에러 메시지 확인:
  ```bash
  adb logcat | grep -i admob
  ```
- AdMob Console에서 앱 등록 상태 확인

### 프로덕션 빌드에서의 광고
- 프로덕션 빌드(`--profile production`)에서는 실제 광고 ID 사용
- AdMob Console에서 광고 단위가 활성화되어 있어야 함

## 4. 테스트 빌드 등록 방법

### 내부 테스트 (Internal Testing)

#### Step 1: EAS 빌드 생성
```bash
# AAB 파일 빌드 (Google Play 제출용)
eas build --profile production --platform android
```

#### Step 2: Google Play Console 설정
1. [Google Play Console](https://play.google.com/console) 접속
2. 앱 생성 → 패키지명 `com.allocate.app`
3. **내부 테스트** 트랙 선택

#### Step 3: 내부 테스트 트랙에 업로드
1. 테스트 → 내부 테스트 → 새 릴리스 만들기
2. EAS 빌드에서 다운로드한 `.aab` 파일 업로드
3. 릴리스 노트 작성 → 검토 시작

#### Step 4: 테스터 등록
1. 내부 테스트 → 테스터 탭
2. 이메일 목록 추가 (최대 100명)
3. 테스터에게 초대 링크 공유

#### EAS Submit 사용 (자동 업로드)
```bash
# Google Play에 자동 제출
eas submit --platform android

# 또는 빌드+제출 한번에
eas build --profile production --platform android --auto-submit
```

> **참고**: 내부 테스트는 Google Play 심사 없이 즉시 배포 가능 (최대 100명)
